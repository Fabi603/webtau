Equality

Identifiers inside validation closure are automatically mapped to a response body. So typing
id.shouldNot == null
will be automatically mapped to response body.id. If you need to access body explicitly use
body.shouldNot == [1, 2, 3] 

http.get("/end-point") {
    id.shouldNot == 0
    amount.should == 30

    list.should == [1, 2, 3]

    object.k1.should == ~/v\d/ 1

    object.should == [k1: 'v1', k3: 'v3'] 2

    complexList.should == ["k1"   | "k2"] { 3
                           ________________
                            "v1"  | "v2"
                            "v11" | "v22" }
}

2matching only specified fields and can be nested multiple times
3matching only specified fields, but number of entries must be exact

Greater/Less/Equal

http.get("/end-point-numbers") {
    id.should beGreaterThan(0)
    price.should beGreaterThanOrEqual(100)
    amount.should beLessThan(150)
    list[1].should beLessThanOrEqual(2)

    id.shouldNot beLessThanOrEqual(0)
    price.shouldNot beLessThan(100)
    amount.shouldNot beGreaterThanOrEqual(150)
    list[1].shouldNot beGreaterThan(2)
}

http.doc.capture("end-point-numbers-matchers")
response
{
  "id": 10,
  "price": 100,
  "amount": 30,
  "list": [
    1,
    2,
    3
  ]
}

Date and Time

You can assert actual string against LocalDate and ZonedDateTime. String will be automatically converted using ISO formatter.
http.get("/end-point-dates") {
    def expectedDate = LocalDate.of(2018, 6, 12)
    def expectedTime = ZonedDateTime.of(expectedDate,
        LocalTime.of(9, 0, 0),
        ZoneId.of("UTC"))

    tradeDate.should == expectedDate
    transactionTime.should == expectedTime
    transactionTime.should beGreaterThanOrEqual(expectedDate)

    paymentSchedule.should contain(expectedDate)
}

http.doc.capture("end-point-dates-matchers")
response

  "paymentSchedule": [
    "2018-06-12",
    "2018-07-10",
    "2018-08-09",
    "2018-09-12"
  ]
}

Mixing Matchers

You can use matchers in place of expected values to build a more complex expectation.
http.get("/end-point-mixed") {
    list.should contain(lessThanOrEqual(2)) 1

    object.should == [k1: 'v1', k3: ~/v\d/] 2

    complexList[0].should == [k1: 'v1', k2: lessThan(120)] 3

    complexList[1].should == [
        k1: notEqual('v1'), 4
        k2: greaterThanOrEqual(120)]

    complexList.should == ["k1"   | "k2"] {
                          ___________________________
                           ~/v\d/ | lessThan(120)
                            "v11" | greaterThan(150) } 5
}

http.doc.capture("end-point-mixing-matchers")
1lessThanOrEqual will be matched against each value
2regular expression match against k3
3lessThen match against k2
4any value but v1
5using matchers as cell values
Note:names greaterThan and beGreaterThan are interchangeable. For better readability use greaterThan version when mixing matchers and beGreaterThan as a top level matcher.
