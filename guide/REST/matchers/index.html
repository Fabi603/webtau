Response Mapping

Identifiers inside validation closure are automatically mapped to a response body.
http.get("/end-point-simple-object") {
    k1.should == 'v1'
}
object response
Copy content
{
  "id": "id1",
  "k1": "v1",
  "k2": "v2"
}
In case of an array response you need to access values using body.
http.get("/end-point-simple-list") {
    body[0].k1.should == 'v1'
}
list response
Copy content
[
  {
    "id": "id1",
    "k1": "v1",
    "k2": "v2"
  }
]

Should and Should Not

Matchers in webtau are triggered with should and shouldNot keywords.
myValue.should contain(10)
myValue.shouldNot equal("hello")
Some matchers have alternative shortcuts.
myValue.should != 10
Additionally shouldBe and shouldNotBe alias keywords are available to make certain matcher combinations easier to read
myValue.shouldBe greaterThan(10)
myValue.shouldBe > 10

Equality

Webtau defines its own set of equality rules to simplify testing.
http.get("/end-point") {
    id.should != 0
    amount.should == 30

    list.should == [1, 2, 3]

    object.k1.should == ~/v\d/ 1

    object.should == [k1: 'v1', k3: 'v3'] 2

    complexList.should == ["k1"   | "k2"] { 3
                           ________________
                            "v1"  | "v2"
                            "v11" | "v22" }
}

Greater/Less/Equal

Standard operations >, >=, <, <= are supported as shortcuts for greaterThan, greaterThanOrEqual, lessThan, and lessThanOrEqual.
http.get("/end-point-numbers") {
    id.shouldBe > 0
    price.shouldBe >= 100
    amount.shouldBe < 150
    list[1].shouldBe <= 2

    id.shouldNotBe <= 0
    price.shouldNotBe < 100
    amount.shouldNotBe >= 150
    list[1].shouldNotBe > 2
}

Contain

Use contain when you cannot rely on order.
http.get("/end-point-list") {
    body.should contain([k1: 'v1', k2: 'v2'])
    body[1].k2.shouldNot contain(22)
}

Date and Time

You can assert actual string against LocalDate and ZonedDateTime. String will be automatically converted using ISO formatter.
http.get("/end-point-dates") {
    def expectedDate = LocalDate.of(2018, 6, 12)
    def expectedTime = ZonedDateTime.of(expectedDate,
        LocalTime.of(9, 0, 0),
        ZoneId.of("UTC"))

    tradeDate.should == expectedDate
    transactionTime.should == expectedTime
    transactionTime.shouldBe >= expectedDate

    paymentSchedule.should contain(expectedDate)
}

Mixing Matchers

You can use matchers in place of expected values to build a more complex expectation.
http.get("/end-point-mixed") {
    list.should contain(lessThanOrEqual(2)) 1

    object.should == [k1: 'v1', k3: ~/v\d/] 2

    complexList[0].should == [k1: 'v1', k2: lessThan(120)] 3

    complexList[1].should == [
        k1: notEqual('v1'), 4
        k2: greaterThanOrEqual(120)]

    complexList.should == ["k1"   | "k2"] {
                          ___________________________
                           ~/v\d/ | lessThan(120)
                            "v11" | greaterThan(150) } 5
}
